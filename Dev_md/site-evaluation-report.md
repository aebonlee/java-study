## 2. 아키텍처 평가

### 2.1 컴포넌트 구조 — 9/10

| 구분 | 파일 수 | 평가 |
|------|---------|------|
| 페이지 컴포넌트 | 72개 | 과정별 디렉토리 분리, lazy loading |
| 허브 페이지 | 5개 | 각 과정별 전용 허브 |
| 공용 컴포넌트 | 6개 | Navbar, Footer, ErrorBoundary, BadgeCard, JavaCodeRunner, CodeBlockCopy |
| Context | 4개 | Theme > Auth > Progress > Badge 계층 구조 |
| 데이터 모듈 | 7개 | lessons, servletLessons, springLessons, practicalLessons, projects, badges, quizzes |

**강점:**
- Suspense + lazy()로 전체 라우트 코드 스플리팅
- Context Provider 계층 구조 명확 (ThemeProvider > AuthProvider > ProgressProvider > BadgeProvider)
- ErrorBoundary로 전역 에러 핸들링
- 데이터와 UI 분리 (data/ 디렉토리)

**개선 가능:**
- 라우트 보호(가드) 미구현 — 인증 필요 페이지에 직접 접근 가능
- ErrorBoundary 기본 UI만 제공 — 에러 로깅, 사용자 피드백 미흡

### 2.2 상태 관리 — 9/10

| Context | 줄 수 | 주요 기능 |
|---------|-------|----------|
| ProgressContext | 284줄 | 레슨 완료, 퀴즈 점수, 코드 실행 횟수, 과정별 진도 |

**개선 가능:**
